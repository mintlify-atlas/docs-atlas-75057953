# Quick start

This guide will help you make your first HTTP requests with Axios. You'll learn the basic patterns for GET and POST requests that you'll use in most applications.

<Note>
  Make sure you've [installed Axios](/installation) before following this guide.
</Note>

## Making a GET request

The simplest way to fetch data with Axios is using `async/await`:

```javascript
import axios from 'axios';

try {
  const response = await axios.get('/user?ID=12345');
  console.log(response);
} catch (error) {
  console.error(error);
}
```

### Using promises

You can also use promise chaining if you prefer:

```javascript
axios.get('/user?ID=12345')
  .then(function (response) {
    console.log(response);
  })
  .catch(function (error) {
    console.log(error);
  })
  .finally(function () {
    // always executed
  });
```

### Passing parameters

Instead of building query strings manually, use the `params` option:

```javascript
axios.get('/user', {
  params: {
    ID: 12345
  }
})
  .then(function (response) {
    console.log(response);
  })
  .catch(function (error) {
    console.log(error);
  });
```

## Making a POST request

Sending data to a server is just as simple:

```javascript
const response = await axios.post('/user', {
  firstName: 'Fred',
  lastName: 'Flintstone'
});
console.log(response);
```

Axios automatically serializes JavaScript objects to JSON and sets the appropriate `Content-Type` header.

## Understanding the response

Every Axios response contains these properties:

```javascript
const response = await axios.get('/user/12345');

console.log(response.data);       // The response body
console.log(response.status);     // HTTP status code (e.g., 200)
console.log(response.statusText); // HTTP status message (e.g., 'OK')
console.log(response.headers);    // Response headers
console.log(response.config);     // The config used for the request
```

## Using the Axios API

You can also pass a configuration object to `axios()` directly:

<Steps>
  <Step title="Send a POST request with config">
    ```javascript
    axios({
      method: 'post',
      url: '/user/12345',
      data: {
        firstName: 'Fred',
        lastName: 'Flintstone'
      }
    });
    ```
  </Step>
  
  <Step title="Send a GET request (default method)">
    ```javascript
    // When no method is specified, GET is used by default
    axios('/user/12345');
    ```
  </Step>
  
  <Step title="Download a file in Node.js">
    ```javascript
    const response = await axios({
      method: 'get',
      url: 'https://bit.ly/2mTM3nY',
      responseType: 'stream'
    });
    
    response.data.pipe(fs.createWriteStream('ada_lovelace.jpg'));
    ```
  </Step>
</Steps>

## Handling errors

Always handle potential errors when making HTTP requests:

```javascript
axios.get('/user/12345')
  .catch(function (error) {
    if (error.response) {
      // The request was made and the server responded with a status code
      // that falls out of the range of 2xx
      console.log(error.response.data);
      console.log(error.response.status);
      console.log(error.response.headers);
    } else if (error.request) {
      // The request was made but no response was received
      console.log(error.request);
    } else {
      // Something happened in setting up the request that triggered an Error
      console.log('Error', error.message);
    }
    console.log(error.config);
  });
```

## Making multiple concurrent requests

Use `Promise.all()` to make multiple requests simultaneously:

```javascript
function getUserAccount() {
  return axios.get('/user/12345');
}

function getUserPermissions() {
  return axios.get('/user/12345/permissions');
}

Promise.all([getUserAccount(), getUserPermissions()])
  .then(function (results) {
    const acct = results[0];
    const perm = results[1];
  });
```

## Creating an instance

For better organization, create an Axios instance with custom configuration:

```javascript
const instance = axios.create({
  baseURL: 'https://some-domain.com/api/',
  timeout: 1000,
  headers: {'X-Custom-Header': 'foobar'}
});

// Now use the instance
const response = await instance.get('/users');
```

<Note>
  Creating instances is useful when you need different configurations for different APIs or want to set up interceptors for specific endpoints.
</Note>

## Request method aliases

Axios provides convenient aliases for common HTTP methods:

<CodeGroup>

```javascript GET
axios.get(url[, config])
```

```javascript POST
axios.post(url[, data[, config]])
```

```javascript PUT
axios.put(url[, data[, config]])
```

```javascript PATCH
axios.patch(url[, data[, config]])
```

```javascript DELETE
axios.delete(url[, config])
```

```javascript HEAD
axios.head(url[, config])
```

```javascript OPTIONS
axios.options(url[, config])
```

</CodeGroup>

<Note>
  When using alias methods, `url`, `method`, and `data` properties don't need to be specified in the config.
</Note>

## Async/await pattern

For cleaner code, wrap your requests in async functions:

```javascript
async function getUser() {
  try {
    const response = await axios.get('/user?ID=12345');
    console.log(response);
  } catch (error) {
    console.error(error);
  }
}
```

<Warning>
  `async/await` is part of ECMAScript 2017 and is not supported in Internet Explorer and older browsers. Use promise chaining if you need to support legacy browsers.
</Warning>
