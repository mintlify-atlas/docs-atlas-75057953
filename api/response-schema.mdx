---
title: AxiosResponse
description: Response object returned by successful Axios requests
---

The `AxiosResponse` interface represents the response returned by Axios when a request is successful. It contains the response data, status, headers, and other metadata.

## Response structure

When using `then`, you will receive the response as follows:

```js
const response = await axios.get('/user/12345');
console.log(response.data);
console.log(response.status);
console.log(response.statusText);
console.log(response.headers);
console.log(response.config);
```

## Response properties

### data

<ResponseField name="data" type="T">
  Response data provided by the server. The type depends on the `responseType` configuration option and can be typed using TypeScript generics
</ResponseField>

```js
const response = await axios.get('/user/12345');
console.log(response.data);
// { id: 12345, name: 'John Doe', ... }
```

### status

<ResponseField name="status" type="number">
  HTTP status code from the server response (e.g., 200, 404, 500)
</ResponseField>

```js
const response = await axios.get('/user/12345');
console.log(response.status);
// 200
```

### statusText

<ResponseField name="statusText" type="string">
  HTTP status message from the server response (e.g., 'OK', 'Not Found')
</ResponseField>

```js
const response = await axios.get('/user/12345');
console.log(response.statusText);
// 'OK'
```

### headers

<ResponseField name="headers" type="RawAxiosResponseHeaders | AxiosResponseHeaders">
  HTTP headers that the server responded with. All header names are lowercase and can be accessed using the bracket notation
</ResponseField>

```js
const response = await axios.get('/user/12345');
console.log(response.headers['content-type']);
// 'application/json'
```

### config

<ResponseField name="config" type="InternalAxiosRequestConfig">
  Configuration object that was provided to axios for the request. This is the merged configuration from defaults and the request-specific config
</ResponseField>

```js
const response = await axios.get('/user/12345');
console.log(response.config.url);
console.log(response.config.method);
```

### request

<ResponseField name="request" type="any">
  The request that generated this response. In Node.js, this is the last ClientRequest instance (in redirects). In the browser, this is an XMLHttpRequest instance
</ResponseField>

```js
const response = await axios.get('/user/12345');
console.log(response.request);
```

## TypeScript support

The AxiosResponse interface supports TypeScript generics for type-safe response handling:

```typescript
interface User {
  id: number;
  name: string;
  email: string;
}

interface RequestData {
  query: string;
}

// AxiosResponse<T = any, D = any, H = {}>
// T: Response data type
// D: Request data type
// H: Custom headers type
const response = await axios.get<User>('/user/12345');
const user: User = response.data; // Type-safe access to response data
```

## Response examples

### JSON response

```js
const response = await axios.get('/api/users/123');

console.log(response);
// {
//   data: { id: 123, name: 'John Doe', email: 'john@example.com' },
//   status: 200,
//   statusText: 'OK',
//   headers: { 'content-type': 'application/json' },
//   config: { ... },
//   request: { ... }
// }
```

### Binary response (arraybuffer)

```js
const response = await axios.get('/api/image.jpg', {
  responseType: 'arraybuffer'
});

console.log(response.data); // ArrayBuffer
console.log(response.headers['content-type']); // 'image/jpeg'
```

### Stream response (Node.js)

```js
import fs from 'fs';

const response = await axios({
  method: 'get',
  url: 'https://example.com/file.pdf',
  responseType: 'stream'
});

response.data.pipe(fs.createWriteStream('file.pdf'));
```

## Accessing response in error handlers

When using `catch`, or passing a rejection callback as the second parameter of `then`, the response will be available through the `error` object:

```js
axios.get('/user/12345')
  .catch(function (error) {
    if (error.response) {
      // The request was made and the server responded with a status code
      // that falls out of the range of 2xx
      console.log(error.response.data);
      console.log(error.response.status);
      console.log(error.response.headers);
    } else if (error.request) {
      // The request was made but no response was received
      console.log(error.request);
    } else {
      // Something happened in setting up the request that triggered an Error
      console.log('Error', error.message);
    }
    console.log(error.config);
  });
```

## Response type variants

The `responseType` configuration option determines the type of `response.data`:

<ResponseField name="arraybuffer" type="ArrayBuffer">
  Response data as an ArrayBuffer (useful for binary data)
</ResponseField>

<ResponseField name="blob" type="Blob">
  Response data as a Blob (browser only, useful for files)
</ResponseField>

<ResponseField name="document" type="Document">
  Response data as an HTML or XML document
</ResponseField>

<ResponseField name="json" type="any">
  Response data as a parsed JSON object (default)
</ResponseField>

<ResponseField name="text" type="string">
  Response data as a plain text string
</ResponseField>

<ResponseField name="stream" type="Stream">
  Response data as a readable stream (Node.js only)
</ResponseField>

<ResponseField name="formdata" type="FormData">
  Response data as FormData
</ResponseField>

```js
// JSON response (default)
const jsonResponse = await axios.get('/api/data');
console.log(jsonResponse.data); // Parsed JSON object

// Text response
const textResponse = await axios.get('/api/data', { responseType: 'text' });
console.log(textResponse.data); // String

// Binary response
const binaryResponse = await axios.get('/api/image', { responseType: 'arraybuffer' });
console.log(binaryResponse.data); // ArrayBuffer
```

## Response headers

Response headers are case-insensitive and can be accessed in multiple ways:

```js
const response = await axios.get('/user/12345');

// Bracket notation (case-insensitive)
console.log(response.headers['content-type']);
console.log(response.headers['Content-Type']);

// Both return the same value
```

Common response headers:

<ResponseField name="content-type" type="string">
  MIME type of the response data
</ResponseField>

<ResponseField name="content-length" type="string">
  Size of the response body in bytes
</ResponseField>

<ResponseField name="content-encoding" type="string">
  Encoding applied to the response body (e.g., gzip)
</ResponseField>

<ResponseField name="cache-control" type="string">
  Caching directives for the response
</ResponseField>

<ResponseField name="set-cookie" type="string[]">
  Cookies set by the server (array of cookie strings)
</ResponseField>

```js
const response = await axios.get('/api/data');

console.log(response.headers);
// {
//   'content-type': 'application/json',
//   'content-length': '1234',
//   'cache-control': 'no-cache',
//   ...
// }
```
